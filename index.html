Heroku Static

Serve your static sites straight from Heroku. That's right — no more nonsense. Simply git push heroku and watch as your site immediately deploys.

If you've never worked with Heroku before, you will need to download both the Heroku CLI and Git. The Heroku Devcenter, in general, will be something you will use quite often.

If you're ready to get started, go ahead and clone this repo on Github and read the simple instructions in the README file.

NOTE: This project was directly inspired by Kenneth Reitz's blog post and Craig Kerstiens' static buildpack.

Questions & Answers

But couldn't I just use Sinatra?

Absolutely. I love Sinatra. However, I know of several people who love designing websites and know how to use Git, but dislike having to use a framework when building a simple site.

Okay, but how much does Heroku cost?

For the most part, Heroku's free. I say "for the most part" because you might need to use a database, at which point you might have to pay some money each month.

Otherwise, welcome to the promised land of awesome web hosting.

I need to use a custom domain, though.

Great. Heroku supports that for free.
